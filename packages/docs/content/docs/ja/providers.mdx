# プロバイダー、モデル、エージェント

OpenChamber が何かを行うには、少なくとも 1 つの AI プロバイダーが接続されている必要があります。このページでは、プロバイダーの接続、モデルの選択、エージェントの調整について説明します。同じプロバイダーを複数回追加して、アカウント、API キー、サブスクリプション、ゲートウェイを分けることができます。

## プロバイダーを接続する

1. **Settings → Providers** を開きます。
2. **Add provider** を開いてプロバイダーを選びます。接続済みのプロバイダーも表示されるため、別のインスタンスを追加できます。
3. `OpenAI 仕事用` のように、アカウントやゲートウェイを識別できる名前を付けます。
4. 別のプロバイダーエンドポイントやカスタムゲートウェイを使う場合は、必要に応じて**ベース URL**を入力します。
5. プロバイダーに応じて、**API key** または **Sign-in (device flow)** で接続します。

各インスタンスは独自の名前、認証情報、ベース URL を保持します。接続済みになると、そのモデルをチャットで使用できます。

インスタンスを切断するには、そのインスタンスを開いて**切断**を選びます。

## インスタンスをプロキシ経由にする

プロキシ経路は、管理対象の API キーインスタンスごとに保存されます。

- **Direct** は、そのインスタンスでプロキシ設定を使用しません。
- **System settings** は OS のプロキシを使用します。Windows では PAC 解決を含み、macOS と Linux では `HTTP_PROXY` または `HTTPS_PROXY` を使い、`NO_PROXY` に従います。
- **Manual proxy** は、そのインスタンスに入力した HTTP(S) プロキシオリジン（例：`http://127.0.0.1:9000`）を使用します。

**ベース URL** はプロバイダー API のエンドポイントを選び、**プロキシ**はそこへ到達するネットワーク経路を選びます。

## OpenAI 互換ゲートウェイを使う

OpenAI Chat Completions に対応している組み込みでないサービスには **OpenAI Compatible** を選び、名前、必須の**ベース URL**（バージョンパスを含む）、API キーを入力します。Frugal Relay では `https://frugalrelay.me/v1` を使います。

OpenChamber は `<ベース URL>/models` からモデル ID を読み込むため、ゲートウェイは OpenAI 互換の `/models` エンドポイントと Chat Completions API を提供する必要があります。

## モデルを選ぶ

作業している場所でモデルを選びます。

- チャットでは、メッセージバーのモデルピッカーを使って、そのセッションのプロバイダーとモデルを設定します
- エージェントごとに、デフォルトモデルを設定します（下記）

## エージェントを設定する

エージェントは、名前付きの設定です。モデル、人格、許可された操作をまとめます。

1. **Settings → Agents** を開きます。
2. エージェントを選ぶか、新しく作成します。
3. 次の項目を編集できます。
   - **description** — エージェントの用途
   - **model** — デフォルトモデル
   - **temperature** — 回答の創造性
   - **prompt** — 常に従う固定指示
   - **tool rules** — 使用してよいツール

## サインイン情報の保存場所

プロバイダーのサインイン情報は OpenChamber ではなく OpenCode に保存されるため、OpenCode CLI と共有されます。同じプロバイダーを複数の場所で設定した場合、最も具体的な設定が優先されます。プロジェクトごとの設定は個人設定を上書きします。
