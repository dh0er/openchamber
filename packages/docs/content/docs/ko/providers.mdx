# 공급자, 모델, 에이전트

OpenChamber가 무언가를 하려면 먼저 최소한 하나의 AI 공급자가 연결되어 있어야 합니다. 이 페이지에서는 공급자 연결, 모델 선택, 에이전트 맞춤 설정을 다룹니다. 계정, API 키, 구독 또는 게이트웨이를 구분하려면 같은 공급자를 여러 번 추가할 수 있습니다.

## 공급자 연결하기

1. **Settings → Providers**를 엽니다.
2. **Add provider**를 열고 공급자를 선택합니다. 연결된 공급자도 계속 표시되므로 다른 인스턴스를 추가할 수 있습니다.
3. `OpenAI 업무용`처럼 계정이나 게이트웨이를 식별할 수 있는 이름을 지정합니다.
4. 다른 공급자 엔드포인트나 사용자 지정 게이트웨이를 사용하려면 선택적으로 **기본 URL**을 입력합니다.
5. 공급자에 따라 **API key** 또는 **Sign-in (device flow)**로 연결합니다.

각 인스턴스는 고유한 이름, 로그인 정보, 기본 URL을 유지합니다. 연결됨으로 표시되면 해당 모델을 채팅에서 사용할 수 있습니다.

인스턴스 연결을 해제하려면 인스턴스를 열고 **연결 해제**를 선택합니다.

## 인스턴스를 프록시로 라우팅하기

프록시 경로는 관리되는 API 키 인스턴스별로 저장됩니다.

- **Direct**는 해당 인스턴스에서 프록시 설정을 사용하지 않습니다.
- **System settings**는 운영 체제 프록시를 사용합니다. Windows에서는 PAC 확인을 포함하며, macOS와 Linux에서는 `HTTP_PROXY` 또는 `HTTPS_PROXY`를 사용하고 `NO_PROXY`를 따릅니다.
- **Manual proxy**는 해당 인스턴스에 입력한 HTTP(S) 프록시 주소(예: `http://127.0.0.1:9000`)를 사용합니다.

**기본 URL**은 공급자 API 엔드포인트를 선택하고, **프록시**는 그 엔드포인트에 도달하는 네트워크 경로를 선택합니다.

## OpenAI 호환 게이트웨이 사용하기

OpenAI Chat Completions를 지원하지만 기본 제공되지 않는 서비스에는 **OpenAI Compatible**을 선택하고 이름, 필수 **기본 URL**(버전 경로 포함), API 키를 입력합니다. Frugal Relay의 경우 `https://frugalrelay.me/v1`을 사용합니다.

OpenChamber는 `<기본 URL>/models`에서 모델 ID를 불러오므로 게이트웨이는 OpenAI 호환 `/models` 엔드포인트와 Chat Completions API를 제공해야 합니다.

## 모델 선택하기

작업하는 위치에서 모델을 선택합니다.

- 채팅에서는 메시지 바의 모델 선택기로 해당 세션의 공급자와 모델을 설정합니다
- 에이전트별로 기본 모델을 설정합니다(아래 참고)

## 에이전트 설정하기

에이전트는 모델, 성격, 허용된 작업을 묶어 이름을 붙인 설정입니다.

1. **Settings → Agents**를 엽니다.
2. 에이전트를 선택하거나 새로 만듭니다.
3. 다음 항목을 편집합니다.
   - **description** — 에이전트의 용도
   - **model** — 기본 모델
   - **temperature** — 답변의 창의성 정도
   - **prompt** — 항상 따르는 상시 지침
   - **tool rules** — 사용할 수 있는 도구

## 로그인 정보가 저장되는 위치

공급자 로그인 정보는 OpenChamber가 아니라 OpenCode에 저장되므로 OpenCode CLI와 공유됩니다. 같은 공급자를 여러 곳에 설정하면 가장 구체적인 설정이 우선합니다. 프로젝트별 설정이 개인 설정을 덮어씁니다.
