# Dostawcy, modele i agenci

Zanim OpenChamber cokolwiek zrobi, potrzebuje co najmniej jednego połączonego dostawcy AI. Ta strona opisuje łączenie dostawcy, wybór modelu i dostosowywanie agentów. Tego samego dostawcę można dodać wiele razy dla osobnych kont, kluczy API, subskrypcji lub bram.

## Połącz dostawcę

1. Otwórz **Settings → Providers**.
2. Otwórz **Add provider** i wybierz dowolnego dostawcę. Połączeni dostawcy nadal są dostępni, więc możesz dodać kolejną instancję.
3. Nadaj instancji nazwę, która wskazuje konto lub bramę, na przykład `OpenAI Praca`.
4. Opcjonalnie wpisz **bazowy adres URL**, aby użyć innego endpointu dostawcy lub niestandardowej bramy.
5. Połącz się za pomocą **klucza API** albo **logowania (przepływ urządzenia)**, zależnie od dostawcy.

Każda instancja zachowuje własną nazwę, dane logowania i bazowy adres URL. Gdy pojawi się jako połączona, jej modele będą dostępne w czacie.

Aby odłączyć instancję, otwórz ją i wybierz **Odłącz**.

## Skieruj instancję przez proxy

Trasa proxy jest zapisywana osobno dla każdej zarządzanej instancji z kluczem API:

- **Bezpośrednio** pomija ustawienia proxy dla tej instancji.
- **Ustawienia systemowe** używają proxy systemu operacyjnego. W Windows obejmuje to rozpoznawanie PAC; w macOS i Linux używane są `HTTP_PROXY` lub `HTTPS_PROXY` z uwzględnieniem `NO_PROXY`.
- **Ręczne proxy** używa adresu HTTP(S) wpisanego dla tej instancji, na przykład `http://127.0.0.1:9000`.

**Bazowy adres URL** wybiera endpoint API dostawcy; **proxy** wybiera trasę sieciową do tego endpointu.

## Użyj bramy zgodnej z OpenAI

Wybierz **OpenAI Compatible** dla niewbudowanej usługi zgodnej z OpenAI Chat Completions. Wpisz nazwę, wymagany **bazowy adres URL** (ze ścieżką wersji) i klucz API. Dla Frugal Relay użyj `https://frugalrelay.me/v1`.

OpenChamber wczytuje identyfikatory modeli z `<bazowy adres URL>/models`, więc brama musi udostępniać zgodny z OpenAI endpoint `/models` oraz API Chat Completions.

## Skonfiguruj agentów

Agent to nazwana konfiguracja — model, osobowość i to, co wolno mu robić.

1. Otwórz **Settings → Agents**.
2. Wybierz agenta lub utwórz nowego.
3. Edytuj dowolne z poniższych:
   - **description** — do czego służy agent
   - **model** — jego domyślny model
   - **temperature** — jak kreatywne są jego odpowiedzi
   - **prompt** — stałe instrukcje, których zawsze przestrzega
   - **tool rules** — których narzędzi może używać

## Gdzie są przechowywane Twoje logowania

Logowania dostawców są przechowywane przez OpenCode, a nie OpenChamber, więc są współdzielone z OpenCode CLI. Jeśli ustawisz tego samego dostawcę w więcej niż jednym miejscu, wygrywa ustawienie najbardziej szczegółowe: ustawienie dla projektu nadpisuje Twoje osobiste.
