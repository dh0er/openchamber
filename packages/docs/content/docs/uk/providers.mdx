# Провайдери, моделі та агенти

Перш ніж OpenChamber зможе щось робити, до нього треба підключити щонайменше одного AI-провайдера. Ця сторінка охоплює підключення провайдера, вибір моделі та налаштування агентів. Того самого провайдера можна додати кілька разів для окремих облікових записів, ключів API, підписок або шлюзів.

## Підключення провайдера

1. Відкрийте **Settings → Providers**.
2. Відкрийте **Add provider** й оберіть будь-якого провайдера. Підключені провайдери залишаються доступними, тому можна додати інший екземпляр.
3. Дайте екземпляру назву, яка позначає обліковий запис або шлюз, наприклад `OpenAI Робота`.
4. За потреби введіть **базову URL-адресу**, щоб використовувати інший endpoint провайдера або власний шлюз.
5. Підключіться за допомогою **ключа API** або **входу (потік пристрою)**, залежно від провайдера.

Кожен екземпляр має власну назву, облікові дані й базову URL-адресу. Коли він відображається як підключений, його моделі стають доступними в чаті.

Щоб відключити екземпляр, відкрийте його та виберіть **Відключити**.

## Маршрутизація екземпляра через проксі

Маршрут проксі зберігається окремо для кожного керованого екземпляра з ключем API:

- **Безпосередньо** оминає налаштування проксі для цього екземпляра.
- **Системні налаштування** використовують проксі операційної системи. У Windows це включає визначення PAC; у macOS і Linux використовуються `HTTP_PROXY` або `HTTPS_PROXY` з урахуванням `NO_PROXY`.
- **Проксі вручну** використовує HTTP(S)-адресу, введену для цього екземпляра, наприклад `http://127.0.0.1:9000`.

**Базова URL-адреса** вибирає endpoint API провайдера; **проксі** вибирає мережевий маршрут до нього.

## Використання шлюзу, сумісного з OpenAI

Оберіть **OpenAI Compatible** для невбудованого сервісу, сумісного з OpenAI Chat Completions. Введіть назву, обов’язкову **базову URL-адресу** (зі шляхом версії) і ключ API. Для Frugal Relay використовуйте `https://frugalrelay.me/v1`.

OpenChamber завантажує ідентифікатори моделей із `<базова URL-адреса>/models`, тому шлюз має надавати сумісний з OpenAI endpoint `/models` та API Chat Completions.

## Вибір моделі

Модель ви обираєте там, де працюєте:

- у чаті використовуйте перемикач моделей у рядку повідомлення, щоб задати провайдера й модель для цієї сесії
- для кожного агента задайте модель за замовчуванням (нижче)

## Налаштування агентів

Агент — це іменоване налаштування: модель, характер та те, що йому дозволено робити.

1. Відкрийте **Settings → Agents**.
2. Оберіть агента або створіть нового.
3. Відредагуйте будь-що з:
   - **description** — для чого призначений агент
   - **model** — його модель за замовчуванням
   - **temperature** — наскільки креативними є його відповіді
   - **prompt** — постійні інструкції, яких він завжди дотримується
   - **tool rules** — якими інструментами він може користуватися

## Де зберігаються ваші входи

Входи провайдерів зберігає OpenCode, а не OpenChamber, тож вони спільні з CLI OpenCode. Якщо ви задаєте одного провайдера в кількох місцях, перемагає найбільш специфічне налаштування: налаштування для проєкту має пріоритет над вашим персональним.
