# 提供商、模型与智能体

在 OpenChamber 能做任何事之前，它至少需要连接一个 AI 提供商。本页介绍如何连接提供商、选择模型以及定制智能体。同一个提供商可以多次添加，以使用不同的账户、API 密钥、订阅或网关。

## 连接提供商

1. 打开 **Settings → Providers**。
2. 打开 **Add provider** 并选择任意提供商。已连接的提供商仍会显示，因此可以添加另一个实例。
3. 为实例指定一个能区分账户或网关的名称，例如 `OpenAI 工作`。
4. 如需使用其他提供商 endpoint 或自定义网关，可以填写可选的**基础 URL**。
5. 根据提供商的支持情况，使用 **API 密钥**或**登录（设备流程）**连接。

每个实例都有自己的名称、凭据和基础 URL。显示为已连接后，其模型即可在聊天中使用。

要断开实例，请打开该实例并选择**断开连接**。

## 通过代理路由一个实例

代理路由按每个使用 API 密钥的托管实例分别保存：

- **直连**忽略该实例的代理设置。
- **系统设置**使用操作系统代理。在 Windows 上包含 PAC 解析；在 macOS 和 Linux 上使用 `HTTP_PROXY` 或 `HTTPS_PROXY` 并遵循 `NO_PROXY`。
- **手动代理**使用为该实例输入的 HTTP(S) 代理地址，例如 `http://127.0.0.1:9000`。

**基础 URL**选择提供商 API endpoint；**代理**选择访问该 endpoint 的网络路径。

## 使用 OpenAI 兼容网关

对于支持 OpenAI Chat Completions 但未内置的服务，请选择 **OpenAI Compatible**，并输入名称、必填的**基础 URL**（含版本路径）和 API 密钥。Frugal Relay 应使用 `https://frugalrelay.me/v1`。

OpenChamber 从 `<基础 URL>/models` 加载模型 ID，因此网关必须提供 OpenAI 兼容的 `/models` endpoint 和 Chat Completions API。

## 选择模型

你在工作的地方选择模型：

- 在聊天中，使用消息栏中的模型选择器为该会话设置提供商和模型
- 按智能体设置默认模型（见下文）

## 设置智能体

智能体是一个命名的配置 — 包括一个模型、一种个性，以及它被允许做的事。

1. 打开 **Settings → Agents**。
2. 选择一个智能体，或创建一个新的。
3. 编辑以下任意项：
   - **description** — 该智能体的用途
   - **model** — 它的默认模型
   - **temperature** — 它的回答有多有创造性
   - **prompt** — 它始终遵循的固定指令
   - **tool rules** — 它可以使用哪些工具

## 你的登录信息存储在哪里

提供商登录信息由 OpenCode 存储，而非 OpenChamber，因此它们与 OpenCode CLI 共享。如果你在多个地方设置了同一个提供商，最具体的设置会生效：按项目的设置会覆盖你的个人设置。
