# Provedores, Modelos e Agentes

Antes de o OpenChamber poder fazer qualquer coisa, ele precisa de pelo menos um provedor de IA conectado. Esta página cobre conectar um provedor, escolher um modelo e personalizar agentes. Você pode adicionar o mesmo provedor várias vezes para usar contas, chaves de API, assinaturas ou gateways separados.

## Conectar um provedor

1. Abra **Settings → Providers**.
2. Abra **Add provider** e escolha qualquer provedor. Os provedores conectados continuam disponíveis para que você possa adicionar outra instância.
3. Dê à instância um nome que identifique a conta ou o gateway, como `OpenAI Trabalho`.
4. Opcionalmente, insira uma **URL base** para usar outro endpoint do provedor ou um gateway personalizado.
5. Conecte-se com uma **chave de API** ou com **Login (fluxo de dispositivo)**, dependendo do provedor.

Cada instância mantém seu próprio nome, credenciais e URL base. Quando ela aparece como conectada, seus modelos ficam disponíveis no chat.

Para desconectar uma instância, abra-a e escolha **Desconectar**.

## Encaminhar uma instância por um proxy

A rota do proxy é salva por instância gerenciada com chave de API:

- **Direto** ignora a configuração de proxy dessa instância.
- **Configurações do sistema** usa o proxy do sistema operacional. No Windows, inclui a resolução de PAC; no macOS e Linux, usa `HTTP_PROXY` ou `HTTPS_PROXY` e respeita `NO_PROXY`.
- **Proxy manual** usa a origem HTTP(S) informada para essa instância, por exemplo `http://127.0.0.1:9000`.

A **URL base** seleciona o endpoint da API do provedor; o **proxy** seleciona a rota de rede usada para alcançá-lo.

## Usar um gateway compatível com OpenAI

Escolha **OpenAI Compatible** para um serviço compatível com OpenAI Chat Completions que não esteja integrado. Insira um nome, a **URL base** obrigatória (incluindo o caminho da versão) e a chave de API. Para o Frugal Relay, use `https://frugalrelay.me/v1`.

O OpenChamber carrega os IDs de modelo de `<URL base>/models`, então o gateway deve oferecer um endpoint `/models` compatível com OpenAI e a API Chat Completions.

## Escolher um modelo

Você escolhe o modelo onde está trabalhando:

- no chat, use o seletor de modelo na barra de mensagens para definir o provedor e o modelo daquela sessão
- por agente, defina um modelo padrão (abaixo)

## Configurar agentes

Um agente é uma configuração nomeada — um modelo, uma personalidade e o que ele tem permissão de fazer.

1. Abra **Settings → Agents**.
2. Escolha um agente ou crie um novo.
3. Edite qualquer um destes:
   - **description** — para que serve o agente
   - **model** — seu modelo padrão
   - **temperature** — quão criativas são suas respostas
   - **prompt** — instruções permanentes que ele sempre segue
   - **tool rules** — quais ferramentas ele pode usar

## Onde ficam seus logins

Os logins de provedores são armazenados pelo OpenCode, não pelo OpenChamber, então são compartilhados com a CLI do OpenCode. Se você definir o mesmo provedor em mais de um lugar, a configuração mais específica vence: uma configuração por projeto sobrescreve a sua pessoal.
